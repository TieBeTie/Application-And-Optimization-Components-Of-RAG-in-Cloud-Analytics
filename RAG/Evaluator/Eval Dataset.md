# Eval Dataset

## Intuition
A RAG evaluation dataset is only as valid as the annotation process that produced it. Synthetic queries are cheap but encode a synthetic notion of "good." Human annotation is the ground truth; synthetic generation scales it.

## Definition

An evaluation dataset $\mathcal{D}$ is a set of triples:

$$\mathcal{D} = \{(q_i,\ a_i^*,\ A_i)\}_{i=1}^{N}$$

where $q_i$ is a query typed by [[Question Taxonomy]], $a_i^*$ is the expected answer, and $A_i$ is a set of **anchors** — verbatim substrings from source documents that must appear in retrieved chunks for $q_i$ to be answerable.

Anchors enable [[Recall@k]] and [[Faithfulness]] without an [[LLM Judge]] on every sample.

Three agents construct and evaluate $\mathcal{D}$ — see [[Evaluation Agents]].

## Properties

### Property 1 — Validity
$\mathcal{D}$ is valid iff [[Evaluation Agents]] independence holds and [[Question Taxonomy]] coverage is sufficient.

Validity is a property of the process that produced $\mathcal{D}$, not of the triples themselves — see [[Construction Protocol]].

### Property 2 — Grounding coverage
For each triple $(q_i, a_i^*, A_i)$, every claim in $a_i^*$ must be traceable to at least one anchor in $A_i$.

**Empirical bound (GaRAGe, ACL 2025).** In a professionally annotated benchmark of 2,366 questions, 97% of answers were grounded in at least one retrieved passage after quality review. A dataset with grounding rate below 90% indicates either anchor extraction errors or questions that cannot be answered from the corpus.

$$\text{grounding rate} = \frac{|\{i : \exists\, s \in \text{system output}_i,\, A_i \cap \text{chunks}(s) \neq \emptyset\}|}{N} \geq 0.90$$

### Property 3 — Synthetic generation is not a valid baseline
A dataset generated by a single LLM is biased toward that model's prior — questions default to factoid + direct + single-hop + in-document, the easiest combination for any retriever. Such a dataset systematically overestimates quality and cannot distinguish retriever architectures.

**Consequence.** Results computed on a fully synthetic dataset are not comparable to results on human-annotated datasets and cannot support claims about system superiority. The current `datasets/queries.jsonl` (10 LLM-generated queries, single document type) does not constitute a valid evaluation baseline and must be replaced.

## Note on Diversity
[[LLM Judge]] Diversity inherits the prior of the Judge model. Treat Diversity as secondary; primary evaluation is [[Recall@k]] and [[Faithfulness]] against anchors $A_i$.
