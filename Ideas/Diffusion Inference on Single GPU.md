# Diffusion Inference on Single GPU

## Проблема

Для инференса диффузионных моделей сейчас нужны две раздельные GPU: одна под саму диффузионную модель, вторая под VAE-энкодер. Это делает локальный запуск дорогим.

## Идея

Запустить всё на одной карте:
- диффузионная модель на GPU
- VAE-энкодер инференсится на CPU с весами в RAM

Либо сжать / перекодировать энкодер так, чтобы оба компонента помещались в одну карту.

## Референс

https://github.com/vllm-project/vllm-omni — vLLM для omni-моделей (энкодеры принимают на вход не только текст). Там есть наработки в этом направлении. Кто-то на GitHub уже пробовал держать энкодер на CPU с весами в RAM.

## Контекст

Специализация — системное программирование, опыт ускорения моделей в компиляторе (Samsung). Цель — написать статью для поступления в докторантуру, выложить в open source.

Прецеденты контрибьюций в open source у МТС есть. Юридических проблем быть не должно, если это не эксклюзивная in-house разработка.
